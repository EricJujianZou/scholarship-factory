# GH-11 — Fetch: `FetchResult` contract (the typed Fetch→Extract seam)

## What shipped

Defines `FetchResult`, the typed hand-off object between the future Fetch
fetcher and Extract. Foundation slice only — the data shape, no fetching, no
adapters, no network.

## Surface changes

### Schemas & data

**`FetchResult`** (`scholarship_factory/fetch.py`) — new pydantic v2
`BaseModel`:

| Field | Type | Notes |
|---|---|---|
| `requested_url` | `str` | URL asked for |
| `final_url` | `str` | URL after redirects; equals `requested_url` when none occurred. This is Extract's `source_url`. |
| `status_code` | `int \| None` | `None` when the request never completed |
| `content_type` | `str \| None` | from the response header |
| `body` | `str \| None` | decoded response text; `None` on failure |
| `fetched_at` | `datetime` | `Field(default_factory=...)`, set to `datetime.now(timezone.utc)` on construction |
| `error` | `str \| None` | short exception summary on failure |
| `ok` | `bool` (derived) | `@computed_field` property, read-only |

`ok` is `status_code is not None and 200 <= status_code < 300 and body is not None`
— not a stored field, so it can't be set to a value that contradicts
`status_code`/`body`.

Exported from `scholarship_factory/__init__.py` (`from .fetch import
FetchResult`, added to `__all__`).

## Behavior & breaking changes

None — purely additive; no existing symbols changed.

- "Success (2xx + body) → `ok=True`; failure (`None`/4xx/5xx/no body) → `ok=False`, may carry `error`; failure with `body=None` valid" → `test_success_result_is_ok`, `test_404_result_is_not_ok`, `test_403_result_is_not_ok`, `test_connection_failure_result_is_valid_and_not_ok`, `test_2xx_with_no_body_is_not_ok`.

- "`fetched_at` populated on construction" → `test_fetched_at_is_populated` (`isinstance(result.fetched_at, datetime)`).
- "Unit tests cover success / 404 / 403 / connection-failure / `ok` derivation, no network" → all present in `tests/test_fetch.py`; no `httpx`/network import anywhere in the diff.

## Review notes

- `ok` deliberately uses `body is not None`, not truthiness — an
  empty-but-successful page (`body=""`) stays `ok=True`, per the locked
  "never fabricate" decision (fetched-but-empty ≠ fetch failed).
- `status_code is not None` guards the range comparison so a failed request
  (`status_code=None`) doesn't raise a `TypeError`.
- The fetcher that actually produces a `FetchResult` (HTTP calls, retries,
  politeness) is a later ticket; this slice is data-shape only.
