# GH-16 — Store-inspection dev CLI

Adds `scholarship_factory/cli.py` + an `sf` console script: a read-only CLI to
inspect the opportunity store before any dashboard exists (pipeline-first
roadmap). Read-only over the existing `OpportunityStore` — no schema change, no
mutation, no network.

- `sf list [--status S] [--db PATH]` — id, title, deadline, status, owner;
  optional status filter; empty store prints nothing (exit 0).
- `sf show <id> [--db PATH]` — full record incl. provenance enums + source spans;
  unknown id -> "not found" on stderr, exit 1.
- db path: `--db`, else `$SF_DB_PATH`, else `./scholarship_factory.db`.

Tests: `tests/test_cli.py` (in-process, temp db, capsys). Built manually (harness
stalled; see engine #92/#93).
