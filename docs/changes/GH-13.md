# GH-13 ‚Äî Parse layer: typed reward/cost money

Adds `scholarship_factory/parse_money.py`: the deterministic money parse layer
(S6). Turns stored verbatim reward/cost strings into a normalized `MoneyValue`
(amount, currency, bound, optional per-period/periods) or `None` when
unparseable. No LLM, no network.

- `parse_money("Up to EUR 7,500")` -> 7500 EUR, `bound=upper`.
- `parse_money("Free")` / `parse_money("$0")` -> 0.
- `parse_money("$5,000/yr for 4 years")` -> total 20000, `per_period=5000`, `periods=4`.
- Unparseable/absent -> `None`.
- `typed_reward`/`typed_cost` mirror `typed_deadlines`: `derived` provenance for a
  computed value, `none` when absent ‚Äî never guessed.

Tests: `tests/test_parse_money.py`. Built manually (harness was stalled; see
engine issues #92/#93).
