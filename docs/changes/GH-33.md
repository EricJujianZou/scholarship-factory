# GH-33 â€” Fetch: response cache (skip refetch within a freshness window)

## What shipped

Session 3 (Fetch) fourth link: a SQLite-backed response cache in the new
`scholarship_factory/cache.py`. `FetchCache` stores successful (`ok`)
`FetchResult`s keyed by URL; `cached_fetch` composes a cache lookup with
any single-arg `fetch_fn` (`fetch_url` or `PoliteFetcher.fetch`) so
reruns mid-batch skip re-downloading bytes within a freshness window.
Failures are never cached, and cached results carry a `from_cache` flag
so callers can tell provenance. Field-level refresh stays out of scope
(Session 8's job).

## Surface changes

### Schemas & data

- New SQLite table `fetch_cache` (`scholarship_factory/cache.py:30`):
  `url` (primary key), `fetched_at`, `status_code`, `content_type`,
  `body`, `final_url`. Created via `CREATE TABLE IF NOT EXISTS`, matching
  the `store.py`/`profile.py` SQLite pattern.
- `FetchResult` (`scholarship_factory/fetch.py:39`) gains
  `from_cache: bool = False`. The computed `ok` field and `error` are
  never persisted as columns â€” `put` writes the six columns explicitly
  rather than round-tripping `model_dump()`.

### CLI / workflows

None â€” library module only.

New public class `FetchCache` (`scholarship_factory/cache.py:19`):
- `FetchCache(db_path, *, clock=lambda: datetime.now(timezone.utc))` â€”
  clock is constructor-injected so staleness tests never sleep for real.
- `put(result: FetchResult) -> None` â€” upserts a row via
  `ON CONFLICT(url) DO UPDATE`.
- `get(url, max_age: timedelta) -> FetchResult | None` â€” returns `None`
  on a miss or on a row older than `max_age`; otherwise reconstructs a
  `FetchResult` with the **original** `fetched_at` and `from_cache=True`.

New module function `cached_fetch(url, *, cache, fetch_fn=fetch_url, max_age=DEFAULT_MAX_AGE)`
(`scholarship_factory/cache.py:88`) â€” cache hit within the window returns
the stored result; miss or stale calls `fetch_fn(url)` with a single
positional argument (preserving composition with `PoliteFetcher.fetch`)
and stores the result only if `result.ok`.

`FetchCache`, `cached_fetch`, and `DEFAULT_MAX_AGE = timedelta(days=1)`
are re-exported from `scholarship_factory/__init__.py`.

## Review notes

- Cache key is the URL as given, not `normalize_apply_url` â€” that
  encodes opportunity-dedup identity, not byte-fetch identity, and would
  wrongly collide two URLs returning different bytes.
- `cached_fetch` calls `fetch_fn(url)` with a single positional argument
  only, by design, so it composes with `PoliteFetcher.fetch` (GH-32)
  without either side knowing about the other. A caller needing custom
  timeouts passes a pre-bound `functools.partial` or a `PoliteFetcher`
  instance.
- No eviction policy beyond staleness â€” the table grows unbounded over
  time, which is per the ticket's scope.
