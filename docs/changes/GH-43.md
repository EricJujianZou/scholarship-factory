# GH-43 — Dashboard: FastAPI read API + single-page opportunity dashboard

## What shipped

The first visible product surface: a local FastAPI service and a single
self-contained HTML page that shows ranked opportunities and lets you edit
the profile they're ranked against. No auth, no build step, single user.
This iteration (2) is a retry that fixes a blocking stored-XSS finding from
iteration 1's review; the API, CLI, and dependency changes are unchanged
from iteration 1.

## Surface changes

### Endpoints / APIs

All in `scholarship_factory/api.py`, via a `create_app(db_path)` factory
(no auth on any route):

- `GET /api/opportunities` — returns `rank()`'s `RankedResults` directly:
  eligible opportunities in rank order plus excluded ones with verdicts and
  reasons, parsed deadline/reward alongside the verbatim stored strings.
- `GET /api/profile` — returns the single `ApplicantProfile` (`owner="me"`),
  creating a default one if none exists.
- `PUT /api/profile` — full-replacement update of
  region/education_level/field_of_study/tags/bio; a subsequent
  `GET /api/opportunities` re-ranks against the new profile.
- `GET /` — serves `scholarship_factory/static/index.html` as
  `text/html`.

### CLI / workflows

- `sf serve [--host 127.0.0.1] [--port 8000] [--db ...]` added to
  `scholarship_factory/cli.py`. Imports `uvicorn` and `.api` lazily inside
  `_cmd_serve` so `sf list`/`sf show` stay import-light. Runs
  `uvicorn.run(create_app(db_path), host, port)`.
- `uvicorn>=0.51.0` added as a dependency (`pyproject.toml`, `uv.lock`).

### Components / pages

- `scholarship_factory/static/index.html` — vanilla-JS dashboard: eligible
  cards (title/org/deadline/reward/apply link), a collapsed `<details>`
  excluded section with verdicts and reasons, and a profile edit form. No
  npm, no framework. Missing facts render as an italic `.missing` span
  ("no deadline found") rather than an invented value.

## Behavior & breaking changes

None — this is new surface, not a change to existing behavior.

## How it was verified

Verified via `tests/test_api.py` (FastAPI `TestClient` against a temp
SQLite db, no live server, no network); `uv run pytest -q` reports 136
passed.

- **Ranked output with parsed + verbatim facts** → `test_opportunities_ranked_with_parsed_and_verbatim`: asserts eligible order `["Eligible Future Grant", "No Deadline Grant"]` matches `rank.py`'s sort key, and that `reward["amount"] == 5000.0` ships alongside the verbatim `reward == "$5,000"` string. Excluded items assert `verdict == "expired"` / `"ineligible"` with non-empty `reasons`.
- **Null deadline, nothing invented** → `test_missing_deadline_serializes_null`: `no_deadline["deadline"] is None` and `opportunity["deadline"] is None` in the JSON; `"no deadline found"` is present in the served `index.html` source (the absent-fact branch of `fmt()`).
- **Profile PUT re-ranks** → `test_profile_update_reranks`: `EU Only Grant` starts in `excluded`, `PUT /api/profile` with `region="Germany"` returns 200 with all five fields updated, and the following `GET /api/opportunities` shows the same opportunity now in `eligible`.
- **Dashboard served correctly** → `test_root_serves_dashboard`: `GET /` returns 200, `text/html` content-type, and the page text references both `/api/opportunities` and `/api/profile`.
- **No auth / TestClient only / green suite** → all five tests above use `TestClient` with no live server or network; the test stage reports 136 passed (up from 135 pre-ticket).
- **Stored-XSS fix (review's blocking finding, this iteration)** → `test_markup_in_stored_fields_is_not_rendered_raw`: seeds a title of `<img src=x onerror="alert(1)">` and `apply_url="javascript:alert(1)"`, asserts the API returns the title byte-identical in JSON (correct — JSON isn't HTML), then asserts on `index.html` source that `escapeHtml` exists, no unescaped `${opp.title}`/`${opp.apply_url}` interpolation remains, and `escapeHtml(opp.title)` / `safeUrl(opp.apply_url)` are present.

## Review notes

- **The XSS fix is verified by source-contract assertions and code review, not rendered-output tests.** `TestClient` executes no JavaScript and no Playwright is available in this session, so nothing here would catch a future refactor that reintroduces a raw interpolation. The escaping itself (`escapeHtml` in `index.html`, folded into `fmt()`'s value branch only — the missing-fact `<span class="missing">` branch stays unescaped by design) and `safeUrl`'s `http:`/`https:` allowlist (degrading a `javascript:` `apply_url` to an unlinked title) were verified by reading the code; a JS-runtime or Playwright test is the natural follow-up, not attempted here per the simplicity rule.
- **Per-request SQLite store construction is intentional.** `OpportunityStore`/`ProfileStore` open their connection in `__init__` with default `check_same_thread=True`; FastAPI runs sync endpoints in a threadpool, so an app-scoped store would raise `ProgrammingError`. Connections aren't explicitly closed (no `close()` on either store) and are reclaimed on GC — acceptable for a single-user local service, worth revisiting if this ever serves more than one user concurrently.
- `PUT /api/profile` is full-replacement (PUT) semantics — a partial body nulls omitted fields.
- Ranking calls `date.today()` with no injectable clock at the API layer; tests use far-past (2020) / far-future (2099) deadlines so verdicts don't rot.
