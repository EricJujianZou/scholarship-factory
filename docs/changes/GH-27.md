# GH-27 — Fetch source adapters: Seed → fetch targets

## What shipped

Session 3 (Fetch) adapter layer: a pure mapping from `Seed` to the target
URL(s) a generic fetcher should pull. Adapters do no network I/O; they only
decide *what* to fetch, per the locked seam (adapters yield targets, one
generic fetcher pulls them). `url` and `devpost` seeds pass through as-is,
`reddit` seeds resolve to the public JSON listing endpoint, and auth-walled
(`instagram`/`x`) or disabled seeds are surfaced as explicit skip outcomes
rather than dropped or raised.

## Surface changes

### Schemas & data

- `FetchTarget` (new, `scholarship_factory/adapters.py`) — `url: str`,
  `seed_type: SeedType`, `label: str | None`.
- `SkipReason` (new) — `str` enum: `DISABLED`, `UNSUPPORTED`.
- `SkippedSeed` (new) — `seed: Seed`, `reason: SkipReason`.
- `AdapterPlan` (new) — `targets: list[FetchTarget]`,
  `skipped: list[SkippedSeed]`; the return type of `targets_for_seeds`.

### CLI / workflows

- `targets_for(seed: Seed) -> list[FetchTarget]` — single-seed adapter.
  `url`/`devpost` pass through unchanged; `reddit` maps to
  `https://www.reddit.com/r/<sub>/new.json?limit=50` via `_subreddit_name`,
  which accepts a bare name, `r/<sub>`, `/r/<sub>`, or a full URL with or
  without trailing slash; disabled or auth-walled seeds return `[]`.
- `targets_for_seeds(seeds: Iterable[Seed]) -> AdapterPlan` — maps a whole
  seed list, splitting results into `targets` and `skipped` (each skip
  tagged `DISABLED` or `UNSUPPORTED`).
- Both re-exported from `scholarship_factory/__init__.py`.

## How it was verified

All criteria verified via `tests/test_adapters.py` (`uv run pytest -q`, 78
passed):

- `url` seed → one target, same URL → `test_url_seed_passes_through`.
- `reddit` bare name and full URL → public JSON target →
  `test_reddit_seed_maps_to_public_json` (5 parametrized input shapes, all
  resolving to `.../r/scholarships/new.json?limit=50`).
- `devpost` seed → its listing URL → `test_devpost_seed_passes_through`.
- `instagram`/`x` seeds → zero targets, explicit `UNSUPPORTED` skip, no
  exception → `test_auth_walled_seeds_are_skipped`.
- `enabled=False` seeds → zero targets, `DISABLED` skip →
  `test_disabled_seed_yields_no_targets`.
- No network: `test_adapters_module_does_no_io` plus manual import check —
  `adapters.py` imports only `collections.abc`, `enum`, `pydantic`, and
  `.seeds`.
- Mixed-list split behavior additionally covered by
  `test_targets_for_seeds_splits_mixed_list`.

## Review notes

- The enabled/auth-walled guard is duplicated between `targets_for` and
  `targets_for_seeds` intentionally, so the single-seed entry point returns
  the correct skip behavior even when called directly (not just through the
  list convenience).
- `_subreddit_name` is string-slicing, not URL parsing — it covers the
  tested input shapes but would need revisiting for inputs like query
  strings appended to a subreddit URL.
- No `wordpress` `SeedType` was added, per the ticket — WordPress listings
  are ordinary `url` seeds and already covered by the `url` branch.
