## Context

This is **Ticket B** (S2 Extract spine); Ticket A (`GH-4`, source-span fields) is already merged — `Opportunity` (`scholarship_factory/models.py:13`) already carries `deadline_source`/`reward_source`/`cost_source`/`source_observed_date` plus a `model_validator` requiring a source whenever `provenance != none`. The package is **flat** (`models.py`, `store.py`, `urls.py`, `__init__.py` re-exports) with pydantic v2; tests live in `tests/` and load fixtures from `tests/fixtures/`. Four raw-HTML fixtures exist (`uwaterloo_grants.html` 11 KB, `oppsforyouth_grants_listing.html` 273 KB, `oppsforyouth_detail.html` 347 KB, `lablab_executorch.html`). Key shaping constraints: **(1)** tests must be fully offline — the Anthropic SDK call must be injectable/stubbed and `anthropic` must **not** be needed at import time (it isn't in `uv.lock` today, and I'm blocked from running `uv`); **(2)** every non-null source span must be a *literal substring of the page text the model saw*, so the extractor must verify spans, not trust them.

## Approach

A three-piece flat module: `clean.py` (deterministic pre-clean, stdlib only), `extract.py` (typed result + the `extract()` contract + LLM→`Opportunity` mapping + a substring-verification gate), and recorded JSON responses under `tests/fixtures/recorded/`. `extract()` takes an injectable `client` (anything exposing `.messages.create`); when `None` it lazily constructs a real `anthropic.Anthropic` inside a `_default_client()` helper, so importing the module and running the tests never touches `anthropic` or the network. The LLM contract uses **Anthropic tool-use as structured output** (a forced tool whose `input_schema` is the response shape); the test stub returns a canned message object mirroring that `tool_use` block, loaded per-fixture from saved JSON. The mapping layer computes provenance from presence (`value present + source verified → quoted`, else `none`) rather than trusting an LLM-emitted enum, and **verifies each claimed source span is a substring of the cleaned text**, nulling any fact whose span is absent — this is where no-fabrication is enforced in code, not just in the prompt. I rejected pulling in `beautifulsoup4`/`lxml` for pre-clean (adds an unlocked dependency for what stdlib `html.parser` handles for our fixtures) and rejected Anthropic's JSON-mode-via-prompt (tool-use gives a real schema and is the SDK's structured-output idiom).

## Risks

1. **Adding `anthropic` breaks the offline sync.** It's not in `uv.lock`; if the build/test env is offline, declaring it as a hard dependency makes `uv run pytest` fail at sync. Mitigation: declare it only as the optional `llm` extra and lazy-import it (steps 6, 5) so the default suite never needs it. If the implementer's `uv run pytest` still tries to resolve it, keep `anthropic` out of `pyproject` entirely and rely solely on the lazy import — the offline tests do not require the package.
2. **"Up to EUR 7,500" vs the page's "Up to €7,500".** AC #3 writes "EUR" but the fixture renders the euro sign (line 65/694). The source span must be a *literal substring*, so the recorded value/span must use the page's "€7,500" form. Mitigation: assert on the numeric "7,500" euro value rather than the literal string "EUR"; if a reviewer reads AC #3 strictly, document that verbatim-substring honesty wins over the AC's paraphrase.
3. **Source-span substring check vs whitespace normalization in pre-clean.** If `clean_html` collapses whitespace differently than the recorded span, a legitimately-present fact gets wrongly nulled. Mitigation: keep pre-clean conservative (collapse runs of whitespace to single spaces, trim) and author recorded spans against the *cleaned* text, not the raw HTML; the step-1 test pins that "Up to $7,500" survives cleaning.
